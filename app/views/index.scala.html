Jira to Sladiator

Description

This is a free site that can be used as a proxy from Jira webhooks to the API of Sladiator.

When this application is called by a webhook it will go back to Jira's API to fetch the required issue information. This information is then converted to Sladiator API information and posted to Sladiator.

Usage

Simply point your Jira webhook to:
http://@(host + routes.Application.webhook("{SLA Token}", "${issue.key}", "{jira host}", "{jira username}", "{jira password}").url)

You can enable the Exclude issue details from the POST to URL option.

Note that we do not store the information in the request in any way. You can check the source code to see how we use it.

An example:
http://@(host + routes.Application.webhook("4bx60a464xd23b5x2ad715cc1fc059be8cc1a9eb", "${issue.key}", "rhinofly.atlassian.net", "api_readonly", "n3E_irTt0a7X*").url)

Troubleshooting

You can use the 'testJiraApi' action to test the connection with Jira.

An example:
http://@(host + routes.Application.testJiraApi("DEMO-1", "rhinofly.atlassian.net", "api_readonly", "n3E_irTt0a7X*"))

For issues and feature requests see this page: https://github.com/Rhinofly/JiraToSladiator/issues
